Gestão da Tecnologia da Informação

Módulo3/Banco_de_Dados



Componente curricular: Banco de Dados

Professor: Braully Rocha da Silva

Ementa:

Níveis de abstração de informações e dados. O modelo conceitual de entidades e relacionamentos (DER/MER). O modelo relacional. Normalização. Integridade Referencial.Indexação em Bancos de Dados. Linguagem SQL (DML e DDL). Estudo de caso: Análise e Projeto.

Objetivos:

O aluno deverá ser capaz de analisar, projetar e construir banco de dados, utilizando uma visão crítica com vista em negócios e segurança da informação.

Competências:

  • Projetar um banco de dados utilizando o Modelo-Relacional e o Modelo-Entidade-Relacionamento;
  • Utilizar a Linguagem SQL para criar, alterar e manipular os dados em um banco de dados relacional;

Conteúdos Programáticos:

  1. Introdução e arquiteturas de bancos de dados;
  2. Modelo-Relacional e suas restrições;
  3. Modelo-Entidade-Relacionamento;
  4. Mapeamento entre Modelos;
  5. Linguagem SQL;
  6. Linguagem de definição de dados;
  7. Linugagem de manipulação de dados;
  8. Indexação e consultas complexas;
  9. Transações em Sistemas de Banco de Dados;

Atividade proposta

Com base nos conteúdos apresentados em sala de aula desenvolver o modelo relacional e o modelo físico de um banco de dados para o problema proposto pela disciplina Matemática Financeira, esse modelo será utilizado por uma aplicação Java SE para executar as seguintes operações: inserir, apagar, alterar e recuperar dados do banco de dados desenvolvido, com base nas informações existentes no banco de dados a aplicação deve recuperar os dados para a execução dos cálculos necessários para a simulação sugerida. O banco de dados também deverá manter um LOG de todas as alterações feitas na base de dados (inclusão, alteração e eliminação de registros), indicando quem fez, quando fez e o que foi feito.
