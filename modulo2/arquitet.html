Gestão da Tecnologia da Informação

Módulo2/Arquitetura_de_Computadores



Componente curricular: Arquitetura e Organização de Computadores

Professora: Kelly Alves Martins de Lima

Ementa:

Estrutura básica de computadores. A Unidade Central de Processamento. Organização de memória e representação de dados.

Objetivos:

Proporcionar ao aluno o entendimento de hardware e de sistemas computacionais, de modo a compreender o funcionamento dos vários módulos que os compõem. Desenvolver uma visão crítica sobre requisitos e desempenho computacional, identificando o impacto de diferentes mecanismos e estruturas computacionais.

Competências:

  • Conhecer o funciomento do hardware dos sistemas computacionais.
  • Entender o funcionamento da estrutura interna de um computador.
  • Projetar requisitos de hardware necessários para atender um parque tecnológico de uma organização.

Conteúdos Programáticos:

  1. Introdução
  2. Componentes de um sistema de computação
    1. Representação das informações
    2. Classificação de sistemas de computação
  3. Subsistemas de memória
    1. Hierarquia de memória
    2. Memória principal
    3. Memória cache
    4. Memória Secundária
  4. Unidade Central de Processamento
    1. Funções básicas da CPU
    2. Instruções de máquina
    3. Funcionamento da CPU
    4. Linguagem de montagem
  5. Representação de dados
    1. Aritmética binária
    2. Tipos de dados
    3. Tipo caractere, lógico e numérico

Atividade proposta

Identificar os requisitos de Hardware necessários para execução do software planejado no componente curricular Programação Orientada a Objetos e configuração do servidor web para geração de logs, em Laboratório de Redes de Computadores, de acordo com o roteiro a seguir: (OBS: Além do processador, memória e HD, deverão ser especificados todos os demais itens de hardware, tais como placas de rede e monitor de vídeo.)

  • Identificar as necessidades de processamento, memória e disco para a criação e execução do software especificado nas disciplinas: Programação Orientada a Objetos, diferenciando requisito para computadores cliente e servidor.
  • Justificar a configuração que foi especificada para a criação e execução do software, considerando: desempenho, velocidade e armazenamento.
  • Identificar as necessidades de processamento, memória e disco para instalação, configuração e o uso das aplicações do servidor web conforme especificado no componente curricular Laboratório de Redes de Computadores.
  • Justificar a configuração que foi especificada para instalação, configuração e o uso das aplicações do servidor web, considerando: desempenho, velocidade e armazenamento.
